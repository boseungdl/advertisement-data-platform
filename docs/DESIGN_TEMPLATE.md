## 1. 단계 × 차원 (본체)

> 각 단계(생성/수집/저장/변환/서빙)에서 6 차원을 훑고 **load-bearing한 것만** 결정으로 남긴다.
> 결정 형식: `결정 — 왜(어느 차원 요구)↑ / 왜 다른 건 아닌가★ / 대가↔`

각 단계에 대해:

```
### <단계명>
- 기능: ____
- 성능·확장: 결정 ____ — 왜↑ ____ / 기각★ ____ / 대가↔ ____
- 신뢰·복구: 결정 ____ — 왜↑ ____ / 기각★ ____ / 대가↔ ____
- 데이터품질: ____
- 보안·거버넌스: ____
- 비용·운영: ____
(그 단계에서 안 걸리는 차원은 "해당없음"으로 명시 — 빈칸과 구분)
```

**단계별 자주 걸리는 차원 힌트** (막히면):
- **생성**: 데이터품질(스키마·drift·결측), 보안(PII), 신뢰(백필 가능?)
- **수집**: 성능(배치/스트림·처리량), 신뢰(전달보장·순서·백프레셔)
- **저장**: 성능(파티셔닝·파일크기), 비용(포맷·보존), 데이터품질(스키마 진화)
- **변환**: 데이터품질(멱등·재처리 정합), 성능(주기·증분/rebuild), 운영(DAG)
- **서빙**: 성능(SQL/API·사전집계·동시성), 비용(스캔량), 보안(권한·마스킹)
