## §1. 도메인 정의 + 핵심 KPI 3개

### §1-A. 도메인 분야 (한 줄 카피)

**[화면]** README §1 도메인 분야 부분 보이도록 스크롤.

**오프닝 멘트**:
> "광고 도메인의 클릭, 노출, 전환 이벤트를 메달리온 아키텍처로 관리하는 데이터 레이크하우스에 대한 발표, 시작하겠습니다."

**호흡 가이드**:
- "광고 도메인의" — 한 박자 쉬고
- "클릭, 노출, 전환" — 세 단어를 끊어서 또박또박 (광고 데이터의 3대 이벤트라는 인상)
- "메달리온 아키텍처로 관리하는" — 한 호흡으로
- "데이터 레이크하우스" — 마무리 강조
- "발표, 시작하겠습니다" — 인사 톤

**보충 멘트 (필요 시)**:
> 🟢 데모와의 차이를 청중이 모를 수 있으니 한 줄로 — "강의 데모는 도메인이 정해지지 않은 generic Iceberg 메달리온이지만, 저는 광고 도메인으로 좁혀서 클릭·노출·전환의 본질을 다뤘습니다."

**예상 청중 반응**:
- 강사 / 현업 데이터 엔지니어 → "메달리온"·"레이크하우스" 단어로 강의 주제와 연결 즉시 인식
- 대표 / 비전공 → "광고 + 클릭/노출/전환" 단어로 대략 그림 잡힘
- 동기 → 여기서는 아직 차별화 안 보일 수 있음 → §1-C 또는 §4에서 풀어줌

### §1-B. 데이터 규모와 패턴

**[화면]** README §1 데이터 규모와 패턴 섹션 (5개 bullet).

**대본**:
> "데이터 규모와 패턴은 다섯 가지로 정의했습니다."
>
> "첫째, 현재는 일 100만 이벤트, 시간당 4만, 초당 12개의 작은 환경입니다."
>
> "둘째, 평균 이벤트 크기 1KB로 일 1GB raw, 한 달 30GB, 1년 365GB가 쌓입니다."
>
> "셋째, 트래픽 패턴은 광고 도메인에 맞춰 직접 정의했습니다. 피크 시간대 평소의 2~3배, 주말 1.5배. 시즌 스파이크는 §7에서 깊이 다루겠습니다."
>
> "넷째, 6개월 내 10배 성장 — 일 1,000만 이벤트가 현실적 목표입니다."
>
> "**다섯째, 1~2년 내 100배까지 — 일 1억 이벤트, 일 100GB.** 이 100배 가정이 이 프로젝트의 모든 아키텍처·스키마 결정을 통과시키는 기준이고, 뒤에 나올 §7 스케일 아웃 시나리오의 출발점입니다."

**호흡 가이드**:
- "다섯 가지로 정의했습니다" — 청중에게 다섯 항목 들을 마음의 준비
- 첫째~넷째는 빠르게 또박또박 (숫자 강조, 부수 정보는 빠르게)
- "**다섯째, 100배**" — 가장 강한 강조. 한 박자 쉬고
- "§7에서 다루겠습니다" — 다음 챕터 호기심 유도

**예상 청중 반응**:
- 강사 / 현업 → 100x 가정의 합리성 + "어디가 깨질까" 사고 시작
- 대표 → "지금 작지만 100배 갈 거다" 비즈니스 메시지
- 동기 → "왜 100x를 가정했나" 의문 → §7에서 답

**스피커 노트**:
- 시즌 스파이크(블프·광군제 3~5배)는 §1-B에서 풀지 말고 §7에서 깊이. 여기서는 "성장 가정 100x" 한 메시지에 집중
- 가이드 정형 템플릿 5항목을 모두 커버한 게 단단한 출발 — "도메인에 맞게 직접 정의"라는 표현이 광고 도메인 사고가 들어갔다는 시그널

### §1-C. 핵심 KPI 3개 — 데이터 엔지니어링 메인

**[화면]** README §1 핵심 KPI 3개 표.

**대본**:
> "이 프로젝트의 성공 기준은 세 가지 숫자로 정했습니다. 모두 데이터 엔지니어링 본질에 닿는 지표입니다."
>
> "첫째, **처리량** — 일 100만 이벤트 처리. 10배 성장하면 일 1,000만까지. 광고 도메인 표준 규모이고 강의 가정과 일치합니다."
>
> "둘째, **신선도** — 늦게 도착하는 전환을 P95 1시간 안에 Gold에 반영합니다. 즉, 100번 측정하면 95번은 한 시간 안에 어제 표가 자동 갱신됨을 보장합니다. **이게 광고 도메인의 본질**입니다 — 전환은 며칠 뒤 도착하기 때문에."
>
> "셋째, **멱등성** — 같은 윈도우를 두 번 돌려도 결과가 비트 단위로 동일합니다. 새벽에 잡이 죽었을 때 사람이 깨어나지 않아도 자동 복구되고, 3개월 백필도 안전합니다. 이 시스템의 **안전벨트**입니다."
>
> "어트리뷰션 두 모델 비교나 음의 ROI 조기 경보 같은 분석 영역 KPI는 메인 파이프라인이 단단해진 다음 Phase 2에서 다루겠습니다. 지금은 데이터 엔지니어링 KPI에 집중합니다."

**호흡 가이드**:
- "세 가지 숫자" — 청중에게 "기능 나열이 아닌 측정 가능한 지표" 시그널
- 각 KPI 이름(**처리량 / 신선도 / 멱등성**) 강조 — 한 박자 쉬고
- "P95"는 풀어주기 — "100번 중 95번"
- "광고 도메인의 본질" — 강조 (이 프로젝트의 핵심 메시지)
- "안전벨트" 비유 — 비전공 청중도 즉각 이해

**예상 청중 반응**:
- 현업 데이터 엔지니어 → "멱등성"·"P95" 단어에 즉시 반응. 다음 슬라이드(§4 Iceberg)에서 답 기대
- 대표 / 비전공 → "안전벨트" 비유로 멱등성의 가치 즉각 이해
- 강사 → 강의 가정(일 100만) 일치 확인 + Phase 2 분리 사고 평가

**스피커 노트**:
- 처리량 / 신선도 / 멱등성 3개는 데이터 엔지니어가 가장 자신 있게 말할 수 있는 KPI 조합
- 분석 영역 KPI(어트리뷰션·ROI)는 Phase 2로 분리한 사고 자체가 발표 임팩트 — "메인 단단해진 후 분석 얹는다" 라는 운영 사고
- "P95 1시간 신선도" + "비트 단위 멱등성" → §4 Iceberg 가치(MERGE 멱등성·Snapshot Isolation)와 자연스럽게 연결
