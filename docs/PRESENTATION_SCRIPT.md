## §1. 도메인 정의 + 핵심 KPI 3개

### §1-A. 도메인 분야 (한 줄 카피)

**[화면]** README §1 도메인 분야 부분 보이도록 스크롤.

**오프닝 멘트**:
> "광고 도메인의 클릭, 노출, 전환 이벤트를 메달리온 아키텍처로 관리하는 데이터 레이크하우스에 대한 발표, 시작하겠습니다."

**호흡 가이드**:
- "광고 도메인의" — 한 박자 쉬고
- "클릭, 노출, 전환" — 세 단어를 끊어서 또박또박 (광고 데이터의 3대 이벤트라는 인상)
- "메달리온 아키텍처로 관리하는" — 한 호흡으로
- "데이터 레이크하우스" — 마무리 강조
- "발표, 시작하겠습니다" — 인사 톤

**보충 멘트 (필요 시)**:
> 🟢 데모와의 차이를 청중이 모를 수 있으니 한 줄로 — "강의 데모는 도메인이 정해지지 않은 generic Iceberg 메달리온이지만, 저는 광고 도메인으로 좁혀서 클릭·노출·전환의 본질을 다뤘습니다."

**예상 청중 반응**:
- 강사 / 현업 데이터 엔지니어 → "메달리온"·"레이크하우스" 단어로 강의 주제와 연결 즉시 인식
- 대표 / 비전공 → "광고 + 클릭/노출/전환" 단어로 대략 그림 잡힘
- 동기 → 여기서는 아직 차별화 안 보일 수 있음 → §1-C 또는 §4에서 풀어줌

### §1-B. 데이터 규모와 패턴

**[화면]** README §1 데이터 규모와 패턴 섹션.

**대본**:
> "현재는 작은 환경입니다 — 일 100만 이벤트, 시간당 4만, 초당 12개 정도. 일 약 1GB raw, 한 달 30GB, 1년 365GB."
>
> "그런데 이 프로젝트의 진짜 포인트는 성장 가정입니다."
>
> "6개월 내 10배(일 1,000만), **1~2년 내 100배(일 1억, 약 100GB/일)**. 이 100배 가정이 이 프로젝트의 모든 아키텍처·스키마 결정을 통과시키는 기준이고, 뒤에 나올 §7 스케일 아웃 시나리오의 출발점입니다."

**호흡 가이드**:
- "현재는 작은 환경" — 한 박자 쉬고 (청중이 "그럼 큰 환경은?"이라 호기심 갖게)
- 일/시간/초 숫자 또박또박, 누적 총량(GB/년)은 빠르게
- "그런데" 강조 — 흐름 전환의 시그널
- "**100배**" — 가장 강한 강조. 일반 청중도 100x는 임팩트 있음
- "§7에서 다루겠습니다" — 다음 챕터 호기심 유도

**예상 청중 반응**:
- 강사 / 현업 → 100x 가정의 합리성 + "어디가 깨질까" 사고 시작
- 대표 → "지금 작지만 100배 갈 거다" 비즈니스 메시지로 받음
- 동기 → "왜 100x를 가정했나" 의문 → §7에서 답

**스피커 노트**:
- 시즌 스파이크(블프·광군제 3~5배)는 §1-B에서 풀지 말고 §7에서 깊이. 여기서는 "성장 가정 100x" 한 메시지에 집중
- 100x 가정의 출처: "광고 도메인은 시즌·M&A·신규 광고주 유입 등으로 단기 폭증이 흔함" 정도까지만 설명. 상세는 §7
- 가이드 §1 정형 템플릿(데이터 규모/패턴/성장)을 그대로 따랐다는 점은 발표에서 굳이 언급 X — 청중이 알아챔
