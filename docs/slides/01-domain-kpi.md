# 발표 슬라이드 1·2 — 도메인과 핵심 KPI

> 1단계 PRD에서 뽑아낸 발표 메모. 슬라이드 디자인은 나중에, 지금은 "무엇을 말할지" 결정.

---

## 슬라이드 1 — 오프닝 (도메인 분야 한 줄)

**제목**: 광고 캠페인 데이터 레이크하우스

**한 줄 카피 (글)**:
> 광고 도메인의 클릭·노출·전환 이벤트를 메달리온 아키텍처로 관리하는 데이터 레이크하우스.

**오프닝 멘트 (말 — 사용자 본인 입에서 자연스러운 표현)**:
> "광고 도메인의 클릭, 노출, 전환 이벤트를 메달리온 아키텍처로 관리하는 데이터 레이크하우스에 대한 발표, 시작하겠습니다."

**비주얼 후보**:
- 광고 데이터 흐름 한 장 (클릭/노출/전환 → 메달리온 3계층 → 분석)
- 또는 단순 제목 슬라이드 (시각화는 다음 슬라이드에서)

**스피커 노트**:
- 오프닝은 30초 안에 청중 머릿속에 "무엇을 만든 프로젝트인지" 박는 것이 목적
- 차별점은 **§3 메달리온 / §4 Iceberg 가치 / §9 멱등성** 자리에서 "페르소나 페인 → 결정"으로 자연스럽게 풀어냄 (가설을 §1에 미리 박지 않음)
- "메달리온 아키텍처" 단어로 강의 흐름과의 연결을 청중(강사 포함)이 즉시 인식

## 슬라이드 2 — 핵심 KPI 3개 (숫자로)

| KPI | 목표 | 의미 |
|---|---|---|
| **1. 처리량** | 일 100만 이벤트 (10x 성장 가정 시 일 1,000만) | 광고 도메인 표준 규모, 강의 가정과 일치 |
| **2. 신선도** | P95 1시간 안에 Gold 반영 | "어제 늦은 전환이 오늘 표에 자동 반영" |
| **3. 멱등성** | 같은 윈도우 두 번 처리 → 결과 동일 | 백필 안전, 새벽 호출 시 재실행 안전 |

**보너스 비즈니스 KPI** (페르소나 기준):
- 캠페인별 ROI / CTR / CVR
- Last-click vs Even-credit 어트리뷰션 비교
- P1 민혁: 일일 보고 1시간 → 10분
- P2 지수: 헬스체크 5분 이내

**스피커 노트**:
- "기능 나열 대신 측정 가능한 숫자로 말씀드린다"
- "P95라는 표현은, 100번 측정했을 때 95번은 1시간 안에 반영된다는 뜻"
- "멱등성은 이 시스템의 안전벨트 — 새벽에 잡이 죽었을 때 재실행해도 데이터가 망가지지 않는다는 약속"

## 발표 시 청중 반응 예측

- **현업 데이터 엔지니어**: 멱등성/백필/Iceberg 매니지먼트 단어에 즉시 반응. 디테일 질문 들어올 가능성 ↑
- **대표/강사**: "왜 Iceberg인가?" "100x 되면 어디 깨지나?" 가정형 질문 가능성 ↑
- **동기**: 차별화 축이 무엇인지 궁금해함

→ 슬라이드 1·2에서 "**광고 도메인 = 늦은 데이터 = Iceberg가 답**" 한 줄을 박아두는 것이 핵심
